Documentation for this program. 

This is the documentation for my cache simulator program. The cache itself is stored as a 3 dimensional python list. The first one is the set, second is the line, and third is the element in each line.

Cache Write. 

This is the function called when we use the command cache write. What it does is first check if it\textquotesingle{}s a hit or miss. On a hit it will update the dirty bit based off of policy and either write to cache or write to both memory and cache. Otherwise on a miss it will either write directly to the address in the ram or call the write allocate function based off of the write miss policy. \mbox{\Hypertarget{namespacecachesimulator_a6726934c4c0fa43d93aeee974d62200b}\label{namespacecachesimulator_a6726934c4c0fa43d93aeee974d62200b}}

Write Miss Allocate. 

Function to do the logic for write miss if we are using the write allocate policy. This is called when we have a write miss. A lot of this logic is similar to having a read miss.
